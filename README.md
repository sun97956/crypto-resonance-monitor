# Crypto Resonance Monitor

多维度共振的加密市场波动率异动监测系统。当价格面异动与杠杆/费率面异动同时发生时，通过 Telegram 推送告警，提示该币接下来波动会放大。

系统不预测价格方向，不执行任何交易，只做"该关注了"的提醒。

---

## 快速开始（给别人用）

只需 3 步即可跑起来：

```bash
# 1. 克隆并安装依赖
git clone https://github.com/sun97956/crypto-resonance-monitor
cd crypto-resonance-monitor
pip install requests

# 2. 配置（复制模板，填入你自己的信息）
cp config.example.json config.json
# 编辑 config.json，把 telegram_chat_id 改成你的 Telegram 用户 ID

# 3. 放好 Bot Token（不要写进 config.json，避免泄露）
# 在同目录创建 .env 文件，内容：
# TELEGRAM_BOT_TOKEN=你的bot_token
```

运行一次验证：

```bash
python monitor.py
```

无命中则静默退出；有共振则 Telegram 收到告警。

> 如何创建 Bot / 获取 chat_id，见 [DEPLOY.md](DEPLOY.md)。定时运行（crontab / 任务计划 / Hermes）也见该文件。

## 当前系统做到了什么

- **监控 50 个主流永续标的**（Binance USDT 永续，市值 ≥ 1 亿美元，剔除稳定币/杠杆代币/交易所平台币/RWA/政治概念币）。
- **四维度 z-score 共振检测**：波动率、成交量（价格面）+ 资金费率、持仓量变化率（结构面）。
- **跨类共振判定**：价格面异常 且 结构面异常 同时发生才告警，避免单一维度噪声。
- **样本外验证有效**：共振信号后 6 小时波动率约为随机时点的 **2.1 倍**、24 小时 **1.6 倍**，约七成信号确实伴随波动放大。
- **零依赖实时推送**：仅用 `requests` + 标准库，直接打 Binance 公开 REST 与 Telegram Bot API，可运行于本机、常驻服务或云实例。
- **平时安静**：阈值 z ≥ 5（极值分位），平均每天约 1.6 条告警，不刷屏。

## 信号方法论

对每个维度计算相对自身近期分布的 z-score：

```
z(x, t) = (x(t) − 均值(窗口)) / 标准差(窗口)
```

| 维度 | 计算内容 | 基线窗口 |
|------|----------|----------|
| 波动率 vol | 5 分钟 K 线实体振幅 | 近 1 天（288 根） |
| 成交量 qvol | 5 分钟成交量 | 近 1 天 |
| 资金费率 fr | FundingRate | 近 7 天（168 根） |
| 持仓变化 oi | 持仓量每小时环比变化 | 近 7 天 |

异常阈值统一为 **|z| ≥ 5**（约 99.999% 分位，极值事件）。

**共振规则：**

```
价格面异常 = |波动率z| ≥ 5  或  (|成交量z| ≥ 5 且 有方向)
结构面异常 = |费率z| ≥ 5  或  |持仓变化z| ≥ 5
共振       = 价格面异常 且 结构面异常
```

波动率与成交量高度同源（放量必伴大实体），两者同触发不构成独立信号，故拆分为价格面与结构面，仅当跨类同时异常才算有效共振。

采用 z-score 而非固定倍数阈值，是因为各币属性差异极大（BTC 与 meme 的波动量级不可比），相对自身基线才能跨币统一衡量"异常程度"。z-score 捕捉的是分布的结构性跳变（费率悄然拉高、持仓骤增等领先征兆），而非对已发生波动的事后描述。

## 币池

- 数据源：CoinGecko 市值排名
- 准入：市值 ≥ 1 亿美元，且存在 Binance USDT 永续
- 剔除：稳定币、贵金属/大宗商品代币、基金/法币代币、杠杆代币（3L/3S）、指数合成币、RWA（如房贷代币）、交易所平台币、新发政治概念币、明显山寨 meme
- 规模：**50 个标的**（见 `universe.json`）

## 回测结果（样本外，参数冻结）

- 样本：50 标的 × 约 56 天，全历史回放
- 无未来函数：z-score 基线仅使用该时刻之前的数据（滚动窗口）
- 参数冻结：阈值 z = 5 由统计极值原理确定，不对样本调参
- 评估标的：信号后波动率相对随机时点的放大倍率（非涨跌方向胜率）

| 指标 | 数值 |
|------|------|
| 共振信号总数 | 92 |
| 平均每日信号数（50 币） | 1.65 |
| 随机基线 6h 波动率 | 2.23% |
| **信号后 6h 波动率** | **4.71%（2.11×）** |
| 随机基线 24h 波动率 | 4.62% |
| **信号后 24h 波动率** | **7.21%（1.56×）** |
| 信号后波动超过随机基线占比 | 69.6% |

共振发生后，后续 6 小时波动为随机时点的 2.1 倍、24 小时 1.6 倍，且约七成的信号确实伴随波动放大。效应方向与幅度一致，表明该信号对"波动放大"具备真实预测力，而非随机噪声。

## 局限

- 系统预测**波动放大**，不预测**价格方向**；价值在于提示"该关注"，而非给出买卖点。
- 样本量（92 次）中等，倍率方向稳定但置信度仍需实时运行累积确认。
- 实时运行依赖主机常驻（进程存活 + 网络连通）；主机离线则监控中断。
- 单交易所（Binance）数据源，标的下架或迁移将导致该标的失效。
- 极端行情下全市场 z-score 同步抬升可能产生批量告警。
